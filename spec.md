## 一、背景与目标

国内居家心脏康复依从性不足 30%，根本原因是患者缺乏安全运动的方法论支撑与持续的行为引导——Heart 的目标是为确诊冠心病（含 PCI/CABG 术后）患者提供 AI 辅助的居家康复管理工具，帮助他们建立可持续的五大处方康复习惯，降低心血管事件风险与再住院率。

---

## 二、目标用户画像

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **术后康复者**（王叔，63 岁，PCI 术后 3 个月） | 已完成手术，出院后缺乏系统康复指导，主治医生只说"慢慢走走"。核心情绪：**恐惧**——怕动了出问题，倾向静养 | "告诉我怎么动才安全，不要让我自己猜" |
| **确诊待管理者**（李阿姨，57 岁，稳定型心绞痛确诊 6 个月） | 长期服药但不知道除了吃药还能做什么。核心情绪：**茫然**——不知道心脏康复是什么概念 | "让我理解为什么要做这些，以及具体怎么做" |

**用户入口筛选原则：**
- 首界面两个按钮：「我已确诊冠心病」/「我在帮家人了解」
- 适用范围：低危 / 中危患者完整使用；高危患者降级为只读模式（见 § 五）

#### F3 首次评估问卷

> 原型：`onboarding用户首次使用路径/assessment-index.html`（问卷主体）· `assessment-summary.html`（评估摘要分叉页）

| # | 功能描述 |
|---|---------|
| F3-1 | 五组字段，≤30 题，预计完成时间 ≤8 分钟；各组之间有过渡语（温和 + 有意义感，非进度提示） |
| F3-2 | **第一组基本信息**：年龄 / 性别 / 身高体重腰围 / 诊断类型（稳定心绞痛 / PCI / CABG / 心梗恢复期）/ 手术距今月数（精确到月，含「24 个月以上」） |
| F3-3 | **第二组心脏功能与高危筛查**：静息心率（可跳过）/ 血压（可跳过）/ LVEF 两步间接问法：「医生有没有提过心脏泵血功能比较弱？」→ 有则填数值或「等我问医生」/ 高危三问（任意「是」→ 立即高危降级）：① 安静时有无胸痛胸闷 ② 近半年有无晕倒/差点晕倒 ③ 有无心力衰竭或严重心律失常诊断 |
| F3-4 | **第三组慢病与用药**：合并症多选（高血压 / 糖尿病 / 高血脂 / 高尿酸 / 都没有）/ β 受体阻滞剂（不确定时弹出常见药名提示）/ 吸烟状态 |
| F3-5 | **第四组功能储备**：VSAQ 简化版必做 → 对应 METs → 驱动初始处方强度 |
| F3-6 | **第五组心理基线快筛**：PHQ-2 + GAD-2（共 4 题）→ 任意≥3 分触发高危标记；完整 PHQ-9 + GAD-7 推迟至注册后第 7 天首次随访 |
| F3-7 | 组内回退：自由翻页；跨组后退弹确认框「返回上一部分会清空当前组的填写，确认吗？」→ 确认则回到上一组第一题 |
| F3-8 | 「更新评估」入口（位于「我」页面）：高影响字段（诊断类型 / 手术月数 / LVEF / 高危三问 / VSAQ）不可直接编辑，需重走相关 F3 组；低影响字段（身高体重腰围 / 吸烟状态 / 慢病标签）可直接编辑 |

#### F2 运动模块

> 原型：`screens/exercise-index.html`（处方主界面）· `screens/exercise-data.html`（F2-16数据可视化）· `flows场景流程内置步骤导航/flow-golden-path.html`（好/一般状态日完整流程）· `flows场景流程内置步骤导航/flow-bad-state.html`（差状态日四选一+抗阻执行）· `flows场景流程内置步骤导航/flow-symptom-warning.html`（F2-3症状锁定流程）

| # | 功能描述 |
|---|---------|
| F2-1 | **危险分层处理**：高危 → 运动 Tab 锁定图标，不进入运动流程；中危 → 相同路径 + 前 18 次强化 HRR 询问 + 每 2 周随访节奏；低危 → 标准流程 |
| F2-2 | **运动页四状态流转**：Check-in → 处方展示 → 执行中（计时 / 自主完成）→ 反馈记录 |
| F2-3 | **Check-in 症状筛查**（步骤一）：胸痛 / 胸闷 / 心慌 / 头晕 / 呼吸困难 / 发热或明显感冒症状 → 任意勾选 → 警告卡 + 锁定当日运动 + 就医建议；次日运动前询问「昨天的不适是否已缓解？」→ 是则解锁，否则维持锁定 |
| F2-4 | **Check-in 状态评估**（步骤二，仅在症状筛查全部为「没有」时触发）：Q1 昨晚睡眠（好 / 一般 / 很差）+ Q2 今天精力（精力好 / 还可以 / 比较累 / 很累）→ **三层状态机（中危/低危统一适用）**：① 好（睡眠好/一般 + 精力好，或睡眠一般 + 精力好）→ 正常处方；② 一般（睡眠一般 AND 精力还可以/比较累；或睡眠好 AND 精力比较累）→ 当日处方降 1 RPE 档，橙色提示，直接进处方；③ 差（睡眠很差 OR 精力很累）→ 差状态日用户四选一 |
| F2-5 | **强度计算双轨**：有手环 → Karvonen 法（HRR）+ RPE 双轨并行；无手环 → RPE 主导，估算心率仅参考；β 阻滞剂用户 → 仅 RPE，心率数据不参与触发逻辑（此药抑制心率反应，Karvonen 法失效） |
| F2-6 | **RPE 四档标签系统（有氧与抗阻分开描述）** 档位名称统一，感受描述按训练类型区分：**有氧（Talk Test 疲劳感）**：轻松（呼吸顺畅，全程很轻松）/ 适中（能说话但不能唱歌，有点喘，目标区间视觉高亮）/ 较累（明显费力，说话需要停顿）/ 很累（几乎说不了话，很吃力）；**抗阻（RIR 留力法）**：轻松（还能再做 6+ 次）/ 适中（还能再做 3–5 次，目标区间视觉高亮）/ 较累（还能再做 1–2 次）/ 很累（几乎力竭） |
| F2-7 | **三层强度调整触发逻辑：** |
| | 层 1（单次）：仅记录，不触发调整 |
| | 层 2（连续 3 次窗口，须在 14 天内完成，否则重置）：**降档**——满足任意一条：3 次中≥2 次 RPE≥15 / 恢复>30 分钟 / 次日 HR>基线+10 次/分 / 1 次轻微不适；**升档**——四项全满足：连续 3 次 RPE≤11 + 无不适 + HR 恢复 + 距上次调整>2 周 |
| | 层 3（每周随访综合）：静息 HR 连续 2 周↓>5 次/分 → 可考虑提升；连续 2 周↑>5 次/分 → 建议降低；6MWT 距离↑≥50m（每月）→ 可递进 |
| | 冲突处理：安全信号优先，降档信号覆盖升档信号 |
| | β 阻滞剂用户：层 2/3 禁用心率相关条件，仅保留 RPE + 6MWT 信号 |
| F2-8 | **调整执行规则**：幅度≤10%，每次只动一个维度（时长 ±5 min 或靶心率 ±5 次/分 或 RPE 目标 ±1 分）；下限 = 首次处方起始强度；连续 3 次降档被接受 → 提示建议与医生确认；用户可选「暂时保持」拒绝建议，系统记录但不强制执行 |
| F2-9 | **运动后反馈屏**：RPE 四档选择（描述按训练类型，见 F2-6）+ **不适两选项**（① 没有不适，感觉良好；② 有些不适 → 展开多选清单：胸痛/头晕/心慌/呼吸困难/其他）+ 心率恢复（有手环路：已恢复 / 心率仍偏高 / 未记录跳过；无手环路：10 分钟内 / 10–30 分钟 / 超过 30 分钟 / 没有特别注意，附手腕测量提示）；有手环用户于运动开始前一次性提示「运动结束后先别关闭手环运动模式，继续佩戴约 10 分钟」 |
| F2-10 | **提交后总结卡片**：运动摘要（类型/时长/RPE 标签/累计次数）+ AI 流式文案（RPE 提交时即后台预生成）+ 下次运动建议（日期/类型/强度）；触发降档时显示「下次强度已小幅调整 ↓ [查看原因]」；AI 调用失败时文案区域显示「今日运动记录已保存」，运动摘要和下次建议仍正常展示 |
| F2-11 | **运动类型库**（C 方案）：步行为默认，用户可切换；每次必推居家运动；器械/场地类随机推；CABG 术后 3 个月内系统过滤上肢中高强度抗阻；抗阻用留力法替代 %1RM（Gismondi 2024 等效）；**抗阻训练 UI 规格**：动作展示方式为文字说明（动作名称 + 姿势要点 + 呼吸提示）+ 引导用户自行搜索视频跟练；组间休息系统倒计时 60–90 秒；多动作一个一个顺序展示，完成一个切下一个；有氧/抗阻/柔韧日程由系统自动安排（抗阻保持非连续日原则），用户不手动排期 |
| F2-12 | **差状态日 / 柔韧呼吸训练**：**差状态日（中危/低危统一）** → 系统不自动分配训练，用户从以下四选一：① 下调强度继续有氧（步行 20 min，RPE 目标降一档）；② 抗阻训练（低强度，Borg≤12）；③ 柔韧训练（推荐，主要肌群静态拉伸 10–15 min，10–30 秒 × 4 次）；④ PLDB 缩唇膈肌呼吸（10 min，吸气 2 s / 缩唇呼气 4 s，约 30 次）；差状态日训练不计入有氧打卡次数；**一般状态日** → 当日处方降 1 RPE 档，不切换训练类型，不跳转差状态日界面，本次不计入层 2 触发计数；**正常运动日** → 热身 5 min（轻度活动+轻度伸展）+ 冷身 5 min（静态拉伸+PLDB），嵌入每次会话，不单独计时 |
| F2-13 | **运动频率**：有氧起始 3 次/周，可进阶至 3–5 次/周；抗阻 2–3 次/周（非连续日）；柔韧每天或≥2–3 次/周；频率调整不由三层触发逻辑驱动，在月度随访或里程碑节点另行提示 |
| F2-14 | **里程碑节点与维持期（形态 C，收口式）**：第 12 次 → 低危（激励卡片）/ 中危（激励 + 强度重评 + 建议线下随访）→ 进入改善期；第 18 次 → 同上强化版；第 90 天 → 结局评估 + 完整康复周期小结（结业仪式）→ 进入维持期；**维持期规则**：随访频率降低（每周→每 2 周），危险分层不变，处方继续更新，用户自主管理为主，产品界面精简（减少主动推送）；不自动降级危险分层 |
| F2-15 | **AI 文案层（原 F4，已合并入 F2）**：规则引擎判断，LLM 生成人话解释；单次触发无对话，上下文独立于康复助手 |
| F2-16 | **数据可视化页**：暖灰配色（#F7F5F1 / #F0EDE8）+ 卡片布局；进度环（本月完成/目标）+ 上月对比 + 平均时长；RPE 趋势折线图（「适中」区间蓝色背景）；静息心率独立折线图（仅手环用户，标「来自手环」标签）；时间轴记录列表（日期/类型/时长/RPE/强度调整标记）；时间维度 Tab：近 2 周 / 近 1 个月 / 全程 |
| F2-17 | **非运动日首页**：运动提示区显示小提示「下次运动建议：明天 / 后天」，不展示运动 CTA 按钮 |
| F2-18 | **一周运动计划视图（运动 Tab 内）**：展示未来 7 天内已规划的运动会话，每条显示日期 / 运动类型（有氧 / 抗阻 / 柔韧）/ 预计时长 / 是否依赖场地或器械（标签形式）；超出一屏时支持滑动；点击任意条目展开当次处方细节；设计意图：减少用户在助手中询问「什么时候运动 / 练什么」的需求 |

## 四、非功能性需求

### 合规边界

- 产品性质：健康管理工具，不构成医疗器械，不做诊断 / 处方行为
- 产品形态：移动端优先 Web App，不开发独立 App（避免医疗器械备案）
- 所有 AI 输出内容须通过 F1b-7「绝对禁止」规则过滤

### 性能

- 首屏加载：FCP < 2.5 s（4G 网络条件下）
- AI 流式文案：RPE 提交时后台预生成，用户阅读运动摘要期间文案就绪，减少可感知等待
- 运动处方页核心内容支持离线查看（用户运动时网络环境不稳定）
- AI 调用超时阈值：5 s，超时后展示降级内容（见 F2-10）

### 安全与隐私

- 用户健康数据加密存储，不对外共享，不用于商业分析
- 账号认证方式 TBD（微信网页授权 / 手机号登录均可行）
- AI System Prompt 中的用户档案数据不在客户端明文暴露

### 可访问性

- 目标用户 40–70 岁：正文字体 ≥ 16 px，关键操作触控区域 ≥ 44 pt
- 核心流程不依赖动画，不做复杂交互动效
- 核心操作步骤 ≤ 3 步（Check-in / 反馈记录 / 随访填写）
- 色彩对比度满足 WCAG AA 标准（对比度≥4.5:1）

### 兼容性

- iOS Safari 14+ / Android Chrome 90+（移动端现代浏览器）
- 支持 iPhone SE（4.7 英寸屏幕）最小尺寸适配

### 成功指标（行为完成率）

- 12 周运动打卡完成率（目标阈值待文献校准后设定）
- 随访问卷完成率（每周/每月）
- 不使用 4 周留存率作为核心指标（慢病产品用户行为模式不同于工具类产品）

## 五、明确不做的事项（Out of Scope）

### 产品形态

- 不开发独立 App（医疗器械备案风险）
- 不做主动推送通知（MVP 阶段不做主动触达）

### 功能模块

- **F7 用药提醒**：无推送能力时功能价值不足，整体取消
- **实时心率监测**：不依赖穿戴设备，心率数据为辅助输入
- **食物数据库 / 餐食记录 / 营养素计算**：用户负担大，依从性差
- **冥想/正念练习功能**：Heart 是心脏康复产品，不成为心理健康 App
- **IMT（吸气肌训练）**：需测定 MIP，超出产品能力范围
- **HRVB（心率变异性生物反馈）**：需专业传感器
- **HIIT 作为主动处方**：仅低危改善期提供原则性科普
- **VT / AT 法强度计算**：临床推荐但居家产品不可用
- **科普知识图谱**：MVP 阶段用内容标签系统替代
- **医生端 / 医生看板**：无医生使用场景
- **用户间社交功能**：无社交设计
- **数据导出功能**：MVP 不做

### 用户范围

- **健康成人（无 CHD 确诊）**：属于预防产品逻辑，不扩展
- **心肌病 / 瓣膜病 / 风湿性心脏病**：康复路径差异大，MVP 不碰
- **心力衰竭患者**：二期扩展
- **高危患者主动康复服务**：降级为只读模式（可访问：饮食 / 康复助手知识问答 / 科普 / 心理科普练习）

### AI 行为边界

- 不做症状诊断
- 不给药物剂量建议
- 不做预后预测
- 不生成含具体数字的运动处方

## 七、用户故事与验收标准

---

### US-01 首次评估完成并获得个性化计划（主流程）

**作为** PCI 术后低危患者，**我希望** 完成首次评估问卷并看到基于我情况的康复计划摘要，**以便** 知道我的起点在哪里以及第一步该怎么做。

#### AC-01-1 正常完成路径

```
Given 用户已通过知情同意，处于 F3 评估第一组起始状态
When  用户依次完成全部 5 组题目，高危三问全部回答「否」，PHQ-2 + GAD-2 任意项得分均 < 3 分
Then  系统完成危险分层计算，结果为低危或中危
      AND 展示评估摘要页，包含：
          - 危险分层标签（通俗表达，不出现「低危」「中危」原文临床术语）
          - 初始运动强度范围（包含运动类型和 RPE 目标档位）
          - 首次运动建议时间窗（今天或明天）
      AND 跳转至功能快速引导页（3–4 张卡片）
```

#### AC-01-2 中断后恢复路径

```
Given 用户完成 F3 前三组后，点击退出小程序
When  用户下次打开小程序，进入首页
Then  首页顶部显示固定 banner，文案为「首次评估未完成 → 点击继续填写」
      AND 运动 Tab、营养 Tab、助手 Tab 均正常可点击进入，不展示任何降级提示
      AND 点击 banner 后，跳转至 F3 第四组第一题（从中断处继续，第 1–3 组数据已保留）
```

#### AC-01-3 高危触发路径

```
Given 用户在 F3 第二组，高危三问第①条「安静时有无胸痛胸闷」回答「是」
When  用户点击「下一题」
Then  系统立即标记用户为高危，不等待后续组题目完成
      AND 评估摘要页标题为「先做好准备」，正文不含「高危」字样
      AND 摘要说明当前可访问功能：饮食 / 康复助手问答 / 科普推送 / 心理科普
      AND 运动 Tab 显示锁定图标，点击后展示说明文案，不进入运动流程
```

### US-02 运动日完成一次有氧训练会话（主流程）

**作为** 低危康复患者（第 4 次运动），**我希望** 按照今日运动处方完成有氧训练并记录反馈，**以便** 系统追踪进度并生成下次建议。

#### AC-02-1 正常运动会话路径

```
Given 今天是运动日，用户打开运动 Tab，当前处于「处方待开始」状态
When  Check-in 步骤一：症状筛查全部选择「没有」
      AND Check-in 步骤二：Q1 睡眠选「好」，Q2 疲劳选「精力好」
Then  进入处方展示页，展示内容包含：
          - 运动类型（文字说明）
          - 时长目标（具体分钟数）
          - 强度标签（轻松 / 适中 / 较累 / 很累 中的一个，不展示 RPE 原始数值）
          - 热身说明（5 分钟）和冷身说明（5 分钟）
```

#### AC-02-2 反馈提交与总结卡片

```
Given 用户完成运动，进入反馈记录页
When  用户选择 RPE =「适中」，症状 =「没有不适」，
      心率恢复（无手环）=「10 分钟内恢复」，点击「提交」
Then  总结卡片展示，包含：
          - 本次运动摘要：类型 / 时长（分钟）/ RPE 标签 / 累计运动次数
          - AI 生成的 2–3 句反馈文字（流式输出）
          - 下次运动建议：日期 + 类型 + 强度标签
      AND 当次运动记录写入运动历史，记录状态为「已完成」
      AND 本次 RPE 数据写入层 2 触发窗口计数
```

#### AC-02-3 差状态日四选一路径（中危/低危统一）

```
Given 用户完成症状筛查（全部「没有」），进入状态评估
When  Q1 睡眠选「很差」OR Q2 疲劳选「很累」（任一触发）
Then  跳过标准处方展示，进入差状态日界面，展示四个方案供用户选择：
          ① 下调强度继续有氧（步行 20 min，RPE 降一档）
          ② 抗阻训练（低强度，Borg ≤ 12）
          ③ 柔韧训练（推荐，主要肌群静态拉伸 10–15 min）
          ④ PLDB 缩唇膈肌呼吸（10 min）
      AND 界面提示「今日训练不计入有氧打卡次数」
      AND 系统不预先选中任何方案，用户主动选择后方可开始
      AND 提供「今天休息」出口
```

### US-03 使用康复助手获取心脏康复知识（主流程）

**作为** 确诊稳定型心绞痛的低危患者，**我希望** 向康复助手询问心脏康复相关知识，**以便** 理解为什么要做这些事并建立康复信心。

#### AC-03-1 通用知识问答

```
Given 用户进入助手 Tab，系统已注入用户档案（低危，初期阶段，无慢病标签）
When  用户输入「为什么心脏病后反而要运动？」
Then  助手从 RAG 知识库检索运动相关内容，返回回复，回复需满足：
          - 包含循证知识（有据可查的说法）+ 通俗解释
          - 不包含具体运动时长、频率或强度数字
          - 不包含「您应该……」类处方性语句
          - 不包含诊断性或预后性判断
```

#### AC-03-2 症状相关问题软阻断

```
Given 用户在助手 Tab 输入「我昨天胸闷，今天能运动吗？」
When  系统识别为意图 C（症状/安全相关）
Then  助手回复建议用户就医确认，不给出「能运动」或「不能运动」的结论
      AND 当日运动模块处方状态不因此次问答发生任何改变
      AND 回复中不出现具体运动建议数字
```

#### AC-03-3 超出知识库范围问题

```
Given 用户输入「我的心脏支架还有多少年寿命？」
When  系统判断为意图 D（超出知识库范围）
Then  助手回复「本模块服务于健康教育和知识科普，不侧重于解答具体问题」
      AND 不尝试推断或估计支架寿命
```

#### AC-03-4 助手处理运动处方级具体问题

```
Given 用户在助手 Tab 输入「我明天应该走多快？走多久？」
When  系统识别为处方级具体问题（意图 B'）
Then  助手回复包含：说明助手提供健康知识科普，具体处方数字在运动页可查看；
      提示用户可继续询问相关知识
      AND 回复不含任何速度 / 时长 / 组数等具体数字
      AND 不引用用户档案中的处方参数数值

Given 用户继续输入「那就告诉我具体走多快」
When  系统识别为多轮追问（触发 F1b-7 升级规则）
Then  助手回复「您的具体计划在运动页面可以查看。」一句话结束
      AND 不再展开任何知识内容
```

### US-05 运动前发现预警症状，运动被锁定（边缘/异常场景）

**作为** 中危康复患者，**我希望** 在 Check-in 发现胸闷时系统立即阻止我运动并给出就医指引，**以便** 在症状未缓解前不进行可能有危险的运动。

#### AC-05-1 症状触发锁定

```
Given 用户在运动日进入运动页，处于 Check-in 状态
When  用户在症状筛查中勾选「胸闷」
Then  页面展示警告卡，文案包含「建议今日停止运动，如症状持续请及时就医」
      AND「开始运动」按钮不可点击（置灰状态）
      AND 当日运动状态标记为「因症状中止」，写入运动历史
      AND 该次记录不计入运动打卡次数
```

#### AC-05-2 次日解锁询问

```
Given 用户当日运动因症状被锁定
When  用户次日首次进入运动页
Then  页面展示询问「昨天的不适感是否已缓解？」（两个选项：已缓解 / 仍有不适）
      AND 选「已缓解」→ 进入正常 Check-in 流程（重新筛查）
      AND 选「仍有不适」→ 维持锁定，再次展示就医建议
```

### US-06 β 阻滞剂用户使用 RPE 主导强度管理（边缘场景）

**作为** 服用美托洛尔（β 受体阻滞剂）的低危患者，**我希望** 系统使用 RPE 而非心率作为强度判断和调整依据，**以便** 不因药物影响心率反应而收到错误的强度调整建议。

#### AC-06-1 处方不展示心率目标

```
Given 用户在 F3 中回答「是否服用 β 受体阻滞剂」为「是」，危险分层为低危
When  用户进入处方展示页
Then  处方页不展示目标心率区间（Karvonen 法相关内容），仅展示 RPE 强度标签
      AND 处方页不展示「今日靶心率区间：xx–xx 次/分」等文案
```

#### AC-06-2 反馈记录：心率恢复作为参考信息不参与计算

```
Given β 阻滞剂用户在反馈记录页填写心率恢复选项
When  用户选择「心率还明显偏高」并提交
Then  该条心率恢复数据记录到用户档案（作为参考数据）
      AND 本次心率恢复结果不触发层 2 降档条件中的心率相关判断
      AND 总结卡片及原因说明不引用心率数据
```

#### AC-06-3 β 阻滞剂用户层 2 降档仅基于 RPE

```
Given β 阻滞剂用户在 14 天内完成 3 次运动，第 1、2 次 RPE =「很累」（≥15），第 3 次 RPE =「较累」（14）
When  系统运行层 2 触发逻辑
Then  触发降档建议（3 次中≥2 次 RPE≥15，满足降档条件）
      AND 降档建议说明中仅引用 RPE 数据，不提及心率数据
```

### US-08 AI 文案生成失败时运动记录仍完整保存（边缘/异常场景）

**作为** 刚完成运动反馈提交的患者，**我希望** 即使后台 AI 文案生成超时，也能看到本次运动摘要和下次建议，**以便** 不丢失运动记录也不陷入空白等待。

#### AC-08-1 AI 调用失败的降级展示

```

```

```

When  用户查看总结卡片
Then  「下次强度已小幅调整 ↓ [查看原因]」提示仍正常展示（由规则引擎触发）

```
